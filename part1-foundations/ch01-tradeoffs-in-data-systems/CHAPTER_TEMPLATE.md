# Chapter 1: Trade-offs in Data Systems Architecture

> "There are no solutions, there are only trade-offs."

## 핵심 개념

- 운영 시스템 vs 분석 시스템
- 트랜잭션 처리와 분석의 특성
- 기록 시스템(System of Record) vs 파생 데이터(Derived Data)
- 분산 시스템 vs 단일 노드 시스템의 트레이드오프

## 요약

### Operational Versus Analytical Systems
- OLTP(Online Transaction Processing): 실시간 트랜잭션 처리
- OLAP(Online Analytical Processing): 대량 데이터 분석

### Data Warehousing
- 데이터 웨어하우스의 목적과 구조
- ETL 프로세스

### Systems of Record and Derived Data
- 기록 시스템: 신뢰할 수 있는 단일 소스
- 파생 데이터: 기록 시스템에서 변환/집계된 데이터

### Cloud Versus Self-Hosting
- 클라우드의 장점: 탄력성, 관리 부담 감소
- 셀프 호스팅의 장점: 통제력, 비용 예측 가능성

### Distributed Versus Single-Node Systems
- 분산 시스템의 복잡성
- 단일 노드로 충분한 경우

### Microservices and Serverless
- 마이크로서비스 아키텍처
- 서버리스 컴퓨팅의 트레이드오프

### Data Systems, Law, and Society
- 데이터 시스템의 사회적 책임
- 법적 규제 고려사항

| OLTP | Online Transaction Processing |

| ETL | Extract, Transform, Load |
| System of Record | 신뢰할 수 있는 데이터의 단일 소스 |
