## 요약

### Relational Model versus Document Model
- 관계형 모델의 탄생과 특징
- NoSQL의 등장 배경
- 문서 모델의 장단점
- 스키마 유연성: Schema-on-Read vs Schema-on-Write
- 데이터 지역성(Data Locality)

### Graph-Like Data Models
- Property Graph 모델
- Triple-Store와 SPARQL
- Cypher 쿼리 언어
- 그래프 모델이 적합한 경우

### GraphQL
- GraphQL의 개념과 특징
- REST API와의 비교
- 장점과 한계

### Event Sourcing and CQRS
- Event Sourcing: 상태 대신 이벤트 저장
- CQRS: Command Query Responsibility Segregation
- 이벤트 소싱의 장단점

### DataFrames, Matrices, and Arrays
- 과학/분석 데이터를 위한 데이터 모델
- 다차원 배열 처리

## 중요 용어

| 용어 | 설명 |
|------|------|
| Document Model | JSON/XML 같은 자기 포함적 문서 구조 |
| Schema-on-Read | 읽을 때 스키마 해석 (문서 DB) |
| Schema-on-Write | 쓸 때 스키마 검증 (관계형 DB) |
| Property Graph | 노드와 엣지에 속성을 가진 그래프 |
| Event Sourcing | 상태 변경을 이벤트로 저장하는 패턴 |
| CQRS | 읽기와 쓰기 모델을 분리하는 패턴 |
